语言模型(LM)

发表于 2022-03-15 更新于 2022-07-18 分类于 知识整理 阅读次数：

前言

在NLP的很多应用中，我们都需要判断一个句子是否合理，比如机器翻译中，需要判断翻译的话是否合理；拼写检查中，正确的词在语句中出现的概率要比错误的词出现的概率更高；在语音识别中，对于同音词的处理。

在机器学习中，对上述需求的做法是计算概率，当一个句子出现的概率较大时，我们可以判断该句更为合理，更容易出现。根据大数定理，当样本量足够大时，概率可以由频率来进行估计，所以我们可以统计频率，再以此估计概率。

最初的语言模型是基于语法规则手工编织的，后来被统计语言模型取代。

后续随着神经网络的发展，提出了神经网络语言模型，核心思想是神经网络的强大的拟合能力，直接拟合出语言模型的概率分布。

在本博客中，主要重点介绍统计语言模型和简单的神经网络语言模型(NNLM)。

根据统计语言模型公式可以知道，一个词的出现概率和它前面的所有词都相关。如果假定一个词出现的概率只与它前面固定数目的词相关，这就是n-gram模型的基本思想，它做了一个n-1阶的Markov的假设，认为一个词出现的概率只与它前面的n-1个词相关。

unigram、bigram、trigram

这样简化，使得统计时需要匹配的词串更短，单个参数的统计变得更加容易，使得参数的总数变少了。

根据n的不同，分为unigram，bigram，trigram，4-gram。

  • unigram

    当n=1时，代表词和周围词是独立的。

  • bigram

    当n=2，就有一个词的出现依赖于前面出现的一个词

  • trigram

    当n=3，一个词的出现依赖于前面出现的两个词

在实际应用中，采用最多的是n=3的三元模型。

在模型效果方面，理论上是n越大，效果越好。但是当n增大到一定程度时，模型效果的提升幅度会变小，模型效果的提升幅度会变小。当n从1到2，从2到3时，模型的效果上升显著，但是n逐渐增大时，效果的提升就不明显了。

这里涉及到了可靠性和可区别性的问题，参数越多，可区别性越好，但同时单个参数的实例变少从而降低了可靠性，因此需要在可靠性和可区分性之间进行折中。

小结

n-gram模型主要在语料中统计各种词串出现的次数以及平滑化处理。概率值计算好后存储下来，当需要计算一个句子的概率时，只需要找到相关的概率参数，将其连乘或相加(log)起来即可。

优点：

  • 涉及到对前面n-1个词条件概率建模，减少了参数量
  • 可解释性强。

缺点：

  • 只能建模到前n-1个词，缺乏长期依赖
  • 当n较大时，模型参数指数增大，还是需要消耗大量的内存空间
  • 得到的共现矩阵稀疏，存在OOV的问题
  • 单纯基于频次统计，容易受到词频的影响，泛化性能差

神经概率语言模型

我们再回想一下上述语言模型的求解方法，通过统计词袋和对应词的出现频率，得到词的出现概率(条件概率)，然后在计算特定句子时，从存储内找到词的出现概率(条件概率)，最后连乘或者取对数求和得到概率值。

在机器学习领域，有一种通用的解决方法，对所考虑的问题建模后先后为其构造一个目标函数，然后对这个目标函数进行优化，从而求得一组最优的参数，然后利用这组最优参数对应的模型来进行预测。

对于统计语言模型，利用最大似然，可以把目标设为单词w的条件概率，直接对条件概率建模，利用神经网络，拟合出w的概率分布。

在训练过程中，词向量也是训练参数，也会进行更新。这里可以看出词向量是语言模型的一个副产物，因为语言模型本身的工作是为了估计给定的一句话有多像人类的话，但从后来的研究发现，语言模型成了一个非常好的工具。

Softmax是一个非常低效的处理方式，需要先计算每个单词的概率，并且还要计算指数，指数在计算机中都是用级数来近似的，计算复杂度很高，最后再做归一化处理。此后很多研究都针对这个问题进行了优化，比如层级softmax、softmax tree。

小结

神经语言模型先给每个词在连续空间中赋予一个向量(词向量)，再通过神经网络去学习这种分布式表征。利用神经网络去建模当前词出现的概率与其前 n-1 个词之间的约束关系。

与n-gram相比，神经概率语言模型优势如下：

  • 词语之间的相似性可以通过词向量来体现，泛化性能更好

举例来说，如果某个（英语）语料中 = “A dog is running in the room”出现了10000次，而 =”A cat is running in the room”只出现了1次。按照n-gram模型的做法，肯定会远大于 。注意, 和的唯一区别在与dog和cat，而这两个词无论是句法还是语义上都扮演了相同的角色，因此
， 和 应该很接近才对。

事实上，由神经概率语言模型算得的 和是大致相等的。原因在于：（1）在神经概率语言模型中假定了“相似的”的词对应的词向量也是相似的；（2)概率函数关于词向量是光滑的，即词向量中的一个小变化对概率的影响也只是一个小变化。这样一来，对于下面这些句子， 只要在语料库中出现一个，其他句子的概率也会相应的增大。

A dog is running in the room
A cat is running in the room
The cat is running in a room
A dog is walking in a bedroom
The dog was walking in the room

  • 基于词向量模型自带平滑化的功能，softmax处理本身就是一个平滑化的过程

在整个神经概率语言模型中：训练时，词向量作为用于构造目标函数的辅助参数，训练完成后，可以视作word embedding矩阵。

缺点：

  • 仅包含了有限的前文信息，于是有了RNNLM。RNNLM的隐状态可以更好的捕捉到前向序列的所有信息(但是也是一种比较弱的存在)，这里没有细讲。

语言模型的评价

我们能够建立语言模型了，一般的评价是我们在训练集上得到语言模型的参数，在测试集里面来测试模型的性能，那么如何去衡量一个语言模型的好坏呢？

比较两个模型A,B好坏，一种外在的评价就是将AB放入具体的任务中，然后分别得到模型的准确率，这种方式当然是最好的方式，但这种方式的缺点是过于耗时，在实际情况中往往需要花费过多时间才能得到结果。另一种方式是使用下面要介绍的困惑度，注意困惑度并不是上述外在评价的一个好的近似，所以一般使用在试点试验上，所谓试点试验就是一个小规模的初步研究，以评估一些性能。

总结

主要总结了一下统计语言模型、n-gram模型、神经概率语言模型的发展，对于ELMO、BERT等语言模型没有涉及到，后续会继续更新相关的模型。

根据语言模型得到的副产物word embeddings矩阵也是后续word2vec的思想来源。

下面撸一下语言模型的历史发展

统计语言模型从概率的角度对语言模型建模，是运用机器学习进行语言建模的开山模型，后续的很多模型都是以此思想得来的，但是统计语言模型有一个很大的缺点就是参数过多，要一次性计算所有词的所有条件概率，需要浪费很多算力。

于是就有了n-gram模型，n-gram模型对统计语言模型的进行了n-1阶的马尔科夫假设，假设当前词只与前面的n-1个词有关，这样减少了大量冗余的计算。但是对于大量的语料来说，当n>3的时候，所需的算力和存储空间也是指数增长的，所以为了减少模型的参数量，就提出了利用神经网络来解决。

神经网络的核心思想还是通过比较小的函数来拟合大的复杂的函数，所以利用神经网络来对所有词的条件概率直接建模，然后进行合理的优化，即可减少参数量。结果证明这种思想是很可靠的，但是相比于语言模型，神经网络的副产物word embeddings矩阵得到了更多的关注，于是就有了后续word2vec。

上述方法都减少了参数量，但是都忽略了长程的依赖性(long distance Dependency)，在NNLM后续，有人做了RNNLM，效果可想而知的比NNLM的好，但是这种long distance dependency是比较weak的，所以就有了ELMO，利用deep的Bi-LSTM在大量语料上做语言模型的预训练。

后续可能会继续更新BERT，ELMO等语言模型，相当于又挖了新坑了。。
